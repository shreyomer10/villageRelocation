Slide 1: Introduction

- Overview of the Village Relocation Management System.
- Purpose: support coordinated relocation planning, field validation, asset tracking and stakeholder reporting.
- The system combines a backend data platform, relational data models, user management, mobile-capable frontend, and a small AI assistant layer.
- AI is one component (~20%) focused on conversational querying, while the larger system handles data integrity, workflows, and field operations.

Slide 2: Proposed System

- Core platform for village relocation data including villages, families, plots, materials, facilities, and employees.
- Backend API built on Flask with MongoDB for scalable document storage and query execution.
- Frontend interfaces for administrators, surveyors, and field teams via the React/Vite app.
- Data workflows include relocation stage tracking, material deliveries, construction verification, and reports.
- AI assistant is integrated for quick conversational access to key metrics and insights.

Slide 3: Important features and Optimizations

- Structured data models for villages, families, plots, materials, meetings, and relocation stages.
- Admin and analytics routes to manage communities, employees, facilities, and material updates.
- Prompt caching and deterministic result handling for demo scenarios.
- Parallel query execution to accelerate dashboard and report generation.
- Secure auth, session persistence, and trace logs for every chat interaction.

Slide 3.1: Offline-First Architecture, Stage Detection using Computer Vision

- Offline-first design enables field devices to keep working during connectivity loss.
- Stage detection can validate construction and relocation progress using computer vision on submitted photos.
- Local processing supports near-real-time verification without waiting for cloud access.
- This improves accuracy for stage-based approvals and construction milestone tracking.

Slide 3.2: Geolocation Verification, Automated Integrity Flagging System

- Geolocation checks confirm submitted village or asset locations against expected site boundaries.
- Integrity flags are raised for inconsistent or suspicious data entries.
- These flags enable early manual review and help prevent data pollution.
- The flagging system supports both field validation and post-hoc audit workflows.
